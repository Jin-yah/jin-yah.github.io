NoAI

View on GitHub →
JavaScript Chrome Extensions API MutationObserver webNavigation API Storage API

Overview

A Chrome extension that gives users control over AI-powered search results on Google. NoAI provides flexible options to either hide or completely block Google's AI-generated content, allowing users to customize their search experience according to their preferences.

Blocker Levels

  • Off: Extension is disabled and does not affect search results
  • Hidden (Default): Hides AI-powered search results from the page without modifying queries; uses DOM monitoring to detect and hide AI overviews in real-time
  • Blocked: Appends -noai to search queries and hides AI results; provides comprehensive blocking though search results may be slightly affected

Technical Implementation

  • MutationObserver Pattern: Implemented real-time DOM monitoring to detect and hide AI overview elements as they appear; observes document changes with childList and subtree options for comprehensive coverage
  • Multi-Language Support: Pattern matching for AI overview headers across 9+ languages including English, German, Japanese, Russian, Chinese, Dutch, Spanish, and Czech
  • Query Modification & UI Cleaning: Automated URL manipulation to append -noai suffix with sophisticated cleanup logic that removes the suffix from search box, address bar, document title, and suggestion dropdowns
  • Flicker Prevention: Page visibility control with document.documentElement.style.visibility during redirects and bfcache restoration handling to ensure smooth user experience
  • Cross-Device Sync: Dual storage architecture using chrome.storage.local for device-specific settings and chrome.storage.sync for blocked count synchronization across devices with periodic alarm-based syncing
  • Smart Tracking: Local buffering of blocked count with periodic sync to cloud storage; increments on both webNavigation events and successful DOM hiding operations
  • Navigation Detection: chrome.webNavigation.onCompleted listener with URL filters for Google search pages (hostContains + pathPrefix pattern matching)

Key Features

  • Slider-based popup UI for intuitive level selection with dynamic descriptions and real-time blocked count display
  • Comprehensive AI content blocking including main search results and "People Also Ask" section AI entries
  • Persistent settings with automatic restoration on browser restart
  • Minimal permissions footprint (storage, webNavigation only)
  • Efficient implementation with negligible performance impact
  • Cross-device blocked count tracking and synchronization

Technical Challenges Solved

  • Handled race conditions between query modification and page rendering to prevent UI flicker
  • Implemented multi-level DOM cleaning to remove suffix from search box, suggestions, titles, and URLs
  • Designed robust pattern matching system to identify AI overviews across different Google page layouts and languages
  • Built periodic sync mechanism to balance real-time tracking with API rate limits

Impact

Empowers users with choice and control over their search experience, addressing growing concerns about AI-generated content in search results. Provides a practical, performant solution for users who prefer traditional search results.

  • Modular Asset System: Support for swappable base model, mouth states, eye states, arm poses, accessories, and face overlays

  • Error Logging: Comprehensive exception handling with automatic error logging to error_log.txt

Technical Implementation

  • Tkinter Canvas Architecture: Multi-layer canvas rendering system with z-index ordering for base, accessories, face overlays, mouth, eyes, and arms
  • Real-Time Audio Processing: sounddevice InputStream callback with NumPy-based amplitude analysis; detects audio levels above 0.10 threshold with 250ms mouth-close delay
  • Event-Driven Animation: Tkinter's after() method for non-blocking animation loops; 4-second blink cycle with 250ms closed-eye duration
  • Global Keyboard Hook: keyboard library integration for system-wide hotkey detection without window focus requirements
  • Dynamic Asset Loading: PIL/Pillow-based image preloading with ImageTk.PhotoImage caching; automatic directory scanning for accessories and face states
  • Transparent Placeholder System: !nothing.png convention for hiding layers; auto-generates 1x1 transparent PNGs if missing
  • State Management: Global state tracking for accessory/face indices, arm animation timing, mouth closure timing, and key press status

Technical Highlights

  • Designed graceful shutdown handling with microphone stream cleanup and keyboard unhooking via WM_DELETE_WINDOW protocol

SleePi-Panda

View on GitHub →
CircuitPython MicroPython Raspberry Pi Pico W USB HID WiFi HTML/CSS

Overview

An IoT device project for the Raspberry Pi Pico W that keeps computers in an active state by periodically sending F13 key inputs via USB HID. Features a charming web interface with an interactive sleepy panda that users can toggle between awake and asleep states, providing a playful solution to prevent computers from entering sleep mode.

Key Features

  • USB HID Keyboard Emulation: Sends F13 key presses every 10 seconds to maintain computer activity without interfering with normal use
  • Web Interface: Hosts a local HTTP server on the Pico W for wireless control from any device on the same network
  • Interactive Panda UI: Custom HTML/CSS interface featuring an animated panda that transitions between awake and sleeping states
  • Toggle & Kill Controls: Web-based buttons to enable/disable the keep-awake function or completely stop the script
  • Fallback Mode: Runs a simplified version without WiFi if network credentials aren't configured
  • LED Status Indicator: Onboard LED reflects the current toggle state for visual feedback

Technical Implementation

  • USB HID Protocol: Utilized CircuitPython's adafruit_hid library to emulate keyboard input; implemented custom F13 keycode (0x68) for non-intrusive wake commands
  • HTTP Server: Built lightweight web server using socketpool with socket reuse (SO_REUSEADDR) and 1-second timeout for responsive request handling
  • WiFi Integration: Automatic connection to configured network using wifi.radio API; prints IP address to boot_out.txt for easy discovery
  • Request Routing: Manual HTTP request parsing with POST endpoint handling for /on, /off, and /stop routes
  • Dual-Mode Operation: Intelligent startup sequence checks for WiFi credentials and switches between server_run() and run_backup() modes
  • State Management: Global variables track toggle state, LED status, and last keypress timestamp with 10-second interval enforcement
  • Template System: Separate HTML files (html_on.html, html_off.html) loaded from filesystem and served based on current state

Technical Highlights

  • Implemented low-level socket programming with recv_into() for efficient buffer management on constrained hardware
  • Designed graceful key release pattern (press → 100ms delay → release) to ensure proper HID event recognition
  • Built responsive event loop balancing HTTP request handling and periodic keypress timing with time.monotonic() tracking
  • Created custom CSS animations for panda sleep/wake transitions with color palette theming
  • Engineered fallback mechanism ensuring device functionality even without network connectivity

Hardware & Software Stack

  • Hardware: Raspberry Pi Pico W (RP2040 dual-core ARM Cortex-M0+ @ 133MHz with WiFi)
  • Firmware: CircuitPython runtime for Python-on-microcontroller development
  • Libraries: adafruit_hid (keyboard emulation), adafruit_requests (HTTP), socketpool (networking)
  • Interface: USB for HID and power, WiFi for web interface hosting

Architecture & Design

  • Event Loop: Main loop handles incoming HTTP connections with non-blocking socket accepts and timeout-based key timing
  • State Synchronization: LED toggling mirrors web interface state for physical feedback
  • Resource Management: Proper socket closing after each request prevents memory leaks on constrained hardware
  • Boot Sequence: 30-second startup delay ensures host computer is ready before HID initialization

Use Cases

  • Preventing computers from auto-sleeping during long downloads or rendering tasks
  • Maintaining active status for remote desktop sessions
  • Keeping presentation systems awake during conferences
  • Development environments requiring continuous uptime
  • Educational tool for learning embedded systems, USB HID, and web servers on microcontrollers

Impact

Demonstrates practical embedded systems development with a focus on user experience. Combines low-level USB HID protocols with modern web technologies to create an accessible, network-controlled hardware solution. Showcases the versatility of CircuitPython for rapid prototyping of IoT devices with both hardware and web interfaces.
